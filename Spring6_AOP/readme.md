# 代理模式
二十三种设计模式中的一种,属于结构型模式。它的作用就是通过提供一一个代理类,让我们在调用目标方法的时
候,不再是直接对目标方法进行调用,而是通过代理类间接调用用。让不属于目标方法核心逻辑的代码从目标方法中
剥离出来一一解耦。调用目标方法时先调用代理对象的方法去,减少对目标方法的调用和打扰,同时让附加功能能够
集中在一起也有利于统一维护。
